SLAM! SETUP GUIDE

Wire up signups to your sheet.

The landing page collects emails. This guide connects it to a Google Sheet you control, in about 10 minutes, using a free Google Apps Script as the middleman. No code on your side — just copy, paste, and click.

Time ~10 min
Cost $0
You'll need A Google account
1

Make a fresh Google Sheet

In a new browser tab, go to:

https://sheets.new

Google creates an empty spreadsheet titled "Untitled spreadsheet." You don't need to add any columns — the script does that for you on the first signup.

  1. Click the title at the top-left ("Untitled spreadsheet") and rename it to something like SLAM! Newsletter Signups.
  2. Leave the sheet open. We'll come back to it.
2

Open the Apps Script editor

From inside the sheet, in the top menu, click:

Extensions  →  Apps Script

A new tab opens with the Apps Script editor — it looks like a simplified code editor. There's a default function called myFunction in the middle of the screen.

  1. Select everything in the editor (⌘+A on Mac, Ctrl+A on Windows) and delete it. The editor should now be empty.
  2. Open the file apps-script.gs from this project, select all of its contents, and copy.
  3. Paste into the empty Apps Script editor.
  4. Save: click the floppy-disk icon  💾  in the toolbar, or press ⌘+S / Ctrl+S.
  5. If it asks for a project name, type something like SLAM Signups and confirm.
What does this script do? When the landing-page form is submitted, the form POSTs the email to this script. The script appends a row to your sheet with: timestamp, email, source, and browser. That's it. No data leaves your Google account.
3

Deploy the script as a web app

Still in the Apps Script editor, click the blue Deploy button in the upper-right, then choose New deployment.

  1. A dialog opens. Next to "Select type," click the gear icon ⚙️ and choose Web app.
  2. Fill in the form exactly like this:
Apps Script — New deployment

New deployment

SLAM signup v1
Me (you@gmail.com) ▾
Anyone ▾
"Anyone" sounds scary — it's fine. This makes the endpoint URL reachable by your landing page (which is a website — it needs to POST to it). It does not make your spreadsheet public. Only people with this exact URL can submit an email. Nobody can read your data.
  1. Click Deploy.
  2. Google asks for permission. Click Authorize access, pick your Google account, and approve.
"Google hasn't verified this app" This warning appears for any personal Apps Script. It's normal — you wrote (well, copied) the script yourself. Click Advanced, then "Go to SLAM Signups (unsafe)", then Allow.
4

Copy the Web app URL

After deploying, a confirmation screen appears with a URL labeled Web app. It looks like:

// Yours will be longer — this is the shape https://script.google.com/macros/s/AKfycb…long random string…/exec
  1. Click the Copy button next to the URL.
  2. Don't close the dialog yet — if you lose the URL, you'll have to re-deploy.
Want to double-check it works? Paste the URL into a new browser tab and hit Enter. You should see: "SLAM! Athletics signup endpoint is live. POST email here." — that means the script is alive and reachable.
5

Paste the URL into the landing page

Switch back to this project and open Landing Page.html.

  1. In the toolbar at the top of the preview, click Tweaks to toggle the tweaks panel on. A floating panel appears in the bottom-right of the page.
  2. Find the Email backend section.
  3. Paste the Apps Script URL into the Apps Script URL field.
  4. Click anywhere outside the field. The URL is saved automatically.
6

Test it

On the landing page, scroll to the hero. Type a test email (e.g. test@example.com) into the form and hit Sign me up.

Duplicates are skipped automatically. If the same email tries to subscribe twice, the script silently ignores it. You won't see duplicate rows in the sheet.
7

Send the newsletter

Once emails are landing in the sheet, you have three good options for actually sending mail:

  1. Mailchimp / Buttondown / Beehiiv / Substack Export the Email column from your sheet as CSV, then import into any of these services. They handle the actual sending, unsubscribe links, and templates.
  2. Gmail mail merge (Google Workspace add-on: Yet Another Mail Merge) Sends directly from your Gmail using the sheet as the source. Free for up to 50/day. Best for small lists where the email comes from "you" personally.
  3. Direct from Apps Script Add a second function to the script that sends a templated email to every row. Ask me and I'll write it — takes about 5 lines.
!

If something's off

  1. The form says "Saved locally" instead of "You're in" The endpoint URL isn't set. Re-paste it into the Tweaks panel.
  2. No row is appearing in the sheet Apps Script caches old versions. Go back to the Apps Script editor → Deploy → Manage deployments → click the pencil ✏️ on your deployment → under "Version," pick New version → Deploy. The URL stays the same but the latest script code is now live.
  3. Console shows a CORS error Expected and harmless. The request goes through anyway (we use no-cors mode). The page assumes success after the POST completes.
  4. I lost the URL Apps Script editor → Deploy → Manage deployments → the URL is shown under your active deployment.
  5. I want to start fresh In the Tweaks panel, clear the URL field. The form falls back to localStorage and you can re-deploy with a new URL anytime.
