Stage 1 — Go live

Take the files in this project and put them on a public URL.

1

Download the project

In this environment (the chat with me), look for the Download or Export option for the project. You'll get a .zip file containing every file we built.

  1. If you don't see a download button, just ask me — I'll generate one for you.
  2. Unzip it on your computer. You should see a folder with files like these:

▸Landing Page.html
▸app.jsx

Don't worry about .jsx files They look like programmer files, but the landing page actually runs them in the browser — nothing to compile, build, or install. Netlify will serve them as-is.
2

Open Netlify Drop

In your browser, go to:

https://app.netlify.com/drop

You'll see a big dashed box that says "Drag and drop your site folder here." That's exactly what it is.

3

Drag the folder in

Drag the unzipped folder (the whole thing — slam-es-athletics) onto the dashed box.

Drag the FOLDER, not the zip Netlify needs the actual folder, not the zip file. Unzip first.
  1. Wait ~20-40 seconds. Netlify uploads the files and gives you a URL like https://chipper-otter-1a2b3c.netlify.app
  2. Click the URL. You should see your landing page, live, on the internet.
That's it — you're live. The form still ships to your sheet (the Apps Script URL is baked into the code). Share that .netlify.app URL with parents to start collecting signups.
4

Claim the site (so you can update it)

The temporary URL works for 24 hours without an account. To keep it permanently — and to update it later — claim it:

  1. On the deploy success page, click Claim this site.
  2. Sign up with your email (or Google).
  3. Done. The site now belongs to your Netlify account.
5

(Optional) Custom domain

If you own a domain like slamesathletics.com:

  1. In Netlify, go to your site → Domain settings → Add custom domain.
  2. Type your domain. Netlify shows you the DNS records to add at your registrar (GoDaddy, Namecheap, etc).
  3. Add those records in your registrar's DNS panel. Wait 5-60 minutes for DNS to propagate.
  4. Netlify auto-issues a free HTTPS certificate. Your site is live at your real domain.
Don't own a domain yet? The free .netlify.app URL works great for testing and even for sharing with parents. Get a real domain when you're ready to print it on flyers.

Stage 2 — Update it

Two flavors of update: content (events, posts, copy) and code (design, layout, new sections). Each has a workflow.

A

Content updates (text changes)

For small text changes — updating an event date, fixing a typo, swapping the description — you have two options:

  1. Come back here, describe the change. Easiest: tell me "change the Q&A about pricing to say $260 instead of $240" and I'll do it. Then redownload + redeploy.
  2. Edit the file yourself, redeploy. Open the downloaded folder, find the relevant file (mostly app.jsx), edit the text, drag the folder back to Netlify (under your site → Deploys → drag to the deploy area). Live in 30s.
Pro move — sheet-as-CMS When you're ready, ask me to build the "Posts & Events tabs in your sheet" version. Then editing content means editing a Google Sheet — no redeploys, no code, no asking me. The site auto-pulls every page load.
B

Design or layout changes

Adding a new section, changing colors, restructuring the page: those need code edits.

  1. Tell me what you want. "Add a coach bios section between hero and What's happening." I build it, you preview here, then redownload and redeploy when happy.
  2. Hire a developer (later). When the site is making real money, a freelance front-end dev can take over. The code is plain HTML + React + CSS — standard stuff. They can pick it up in an hour.
C

Backend updates (the Google Sheet pipeline)

If you ever change which Google Sheet collects emails, or want to add more fields (name, kid's age, etc):

  1. Re-open the Apps Script editor (Extensions → Apps Script from your sheet).
  2. Edit apps-script.gs in there.
  3. Click Deploy → Manage deployments → pencil → New version → Deploy. The URL stays the same.

No need to redeploy the website — only the Apps Script changes.

Things that might trip you up.

?

Common gotchas

  1. I deployed, the page loads, but the email form doesn't work. Check the browser console (right-click → Inspect → Console). If you see a CORS error, that's expected and harmless — the request goes through. If you see "no endpoint configured," the URL got lost during redownload — re-add it in app.jsx under signupEndpoint.
  2. The logo is broken or missing on the live site. You probably uploaded only some files. Re-upload the entire folder — especially the assets/ and fonts/ subfolders.
  3. The Tweaks panel won't open on the live site. Expected. Tweaks is only for the design environment here; once deployed, it's hidden by default. To change the hero background or endpoint URL on the live site, edit app.jsx directly in the TWEAK_DEFAULTS block.
  4. I want to roll back to a previous version. Netlify keeps every deploy. In your site → Deploys → click any older deploy → Publish deploy. Reverts instantly.
